Tienda web TechShop

● desplegado HTML CSS JavaScript Apache

Sitio web de una tienda de tecnología desarrollado con HTML, CSS y JavaScript: tres páginas con estética coherente y validación de formulario por DOM. Lo curioso: está desplegada de verdad sobre el servidor Apache del proyecto de sistemas.

Stack
HTML · CSS · JS (vanilla)
Páginas
Portada · Producto · Contacto
Despliegue
Apache (capsulecorp)
Módulo
Aplicaciones web

Objetivos

  • Crear tres páginas (portada, producto y contacto) con una estética consistente.
  • Mostrar un producto con su imagen, precio, descripción y botón de carrito.
  • Construir un formulario de contacto con validación de campos por DOM.
  • Desplegar el sitio en un servidor web real.

Portada

Página de inicio con cabecera, barra de navegación hacia las otras dos páginas, productos destacados con descuentos y una sección de ventajas de la tienda. Es la base que define la identidad visual del sitio.

Ficha de producto

Página de ejemplo de un producto con imagen, nombre, precio, descripción, selector de cantidad y un botón "Agregar al carrito" que muestra una confirmación. Incluye pestañas de descripción, especificaciones y opiniones.

Formulario de contacto

El requisito clave era validar que todos los campos estén rellenos usando el DOM. Con getElementById se comprueba nombre, email y mensaje antes de enviar, marcando en rojo los campos incorrectos:

validacion.js
document.getElementById('formulario-contacto')
  .addEventListener('submit', function (e) {
    e.preventDefault();
    var valido = true;

    var nombre = document.getElementById('nombre').value;
    var email  = document.getElementById('email').value;

    // El nombre no puede estar vacío
    if (nombre.trim() === '') {
      document.getElementById('nombre').classList.add('error');
      valido = false;
    }
    // ...se validan también email y mensaje

    if (valido) {
      mostrarMensajeExito();
    }
  });

Despliegue real

En lugar de quedarse en local, esta web se subió por SCP al servidor del proyecto de sistemas y se sirve desde un VirtualHost de Apache (capsulecorp.local). Es un buen ejemplo de cómo encajan dos módulos distintos: la web del módulo de aplicaciones corriendo sobre la infraestructura del módulo de sistemas.

🔗

Esta web es la que se aloja en el servidor capsulecorp a través de su VirtualHost de Apache.

Conclusión

Un proyecto más ligero que los de sistemas y redes, pero que me sirvió para repasar HTML/CSS/JS y, sobre todo, para ver el ciclo completo: desarrollar la web y luego desplegarla en un servidor propio. Esa unión entre desarrollo y operación es justo lo que me atrae de DevOps.
