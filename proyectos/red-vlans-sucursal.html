// Administración de redes

Red local segmentada con VLANs

● completado Cisco VLAN VTP STP DHCP

Diseño e implementación en Cisco Packet Tracer de la red de una sucursal con segmentación por departamentos, un switch de capa 3 como núcleo, servicios de red centralizados y seguridad básica en todos los dispositivos.

Herramienta
Cisco Packet Tracer
Núcleo
Switch L3 (CORE_RED)
VLANs
10 · 20 · 30 · 99
Servicios
DHCP · DNS · HTTP

Topología

Un router principal conectado a un switch de capa 3 (núcleo), del que cuelgan dos switches de acceso con enlace redundante y la red de servidores. Cada departamento tiene su propia VLAN.

Objetivos

  • Segmentar la red por departamentos con VLANs gestionadas mediante VTP.
  • Configurar el enrutamiento inter-VLAN en el switch de capa 3.
  • Centralizar DHCP y ofrecer servicios DNS/HTTP.
  • Aplicar redundancia con Spanning Tree y seguridad básica en los equipos.

VLANs y VTP

El switch de capa 3 actúa como servidor VTP y propaga las VLANs; los switches de acceso son clientes, así no hay que crear las VLANs en cada uno.

CORE_RED · IOS
CORE_RED(config)# vtp mode server
CORE_RED(config)# vtp domain REDPRUEBA
CORE_RED(config)# vlan 10
CORE_RED(config-vlan)# name ADMINISTRACION
CORE_RED(config)# vlan 20
CORE_RED(config-vlan)# name VENTAS
CORE_RED(config)# vlan 30
CORE_RED(config-vlan)# name SOPORTE_TECNICO

! Los switches de acceso → cliente VTP
SWITCH_DISTRIBUCION_1(config)# vtp mode client

Enrutamiento inter-VLAN

El switch de capa 3 enruta entre VLANs mediante interfaces virtuales (SVI). Como el servidor DHCP está en otra VLAN, se usa ip helper-address para reenviarle las peticiones.

CORE_RED · SVI + DHCP relay
CORE_RED(config)# ip routing

CORE_RED(config)# interface vlan 10
CORE_RED(config-if)# ip address 192.168.10.1 255.255.255.0
CORE_RED(config-if)# ip helper-address 10.0.0.2
CORE_RED(config-if)# no shutdown

Servicios de red

En la red de servidores (VLAN 99) se ubica el servidor DHCP, que asigna direcciones a todas las VLANs, y un servidor separado con DNS y HTTP (resolviendo www.cisco.com como prueba). Los pools dejan al menos 25 direcciones libres para asignación manual.

Redundancia con Spanning Tree

Para que el switch núcleo sea el root bridge de todas las VLANs (y el tráfico circule de forma óptima por el enlace redundante), se le asigna la prioridad más baja:

CORE_RED · STP
CORE_RED(config)# spanning-tree vlan 10 priority 4096
CORE_RED(config)# spanning-tree vlan 20 priority 4096
CORE_RED(config)# spanning-tree vlan 30 priority 4096
CORE_RED(config)# spanning-tree vlan 99 priority 4096

Seguridad de los dispositivos

Configuración mínima de seguridad en routers y switches: contraseña de modo privilegiado, de consola y de acceso remoto (vty), cifrado de contraseñas, banner de advertencia y guardado de la configuración.

IOS · seguridad básica
Router(config)# enable secret cisco123
Router(config)# line console 0
Router(config-line)# password cisco1234
Router(config-line)# login
Router(config)# line vty 0 4
Router(config-line)# password cisco1234
Router(config-line)# login
Router(config)# service password-encryption
Router(config)# banner motd #SOLO PERSONAL AUTORIZADO#
Router# copy running-config startup-config

Conclusión

Un proyecto de redes muy completo donde se ven en conjunto la segmentación, el enrutamiento de capa 3, los servicios centralizados y la redundancia. Entender cómo "hablan" entre sí las VLANs y por qué se centraliza el DHCP es básico para cualquier infraestructura sobre la que después corre DevOps.
