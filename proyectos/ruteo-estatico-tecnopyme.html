Ruteo estático multi-sede

● completado Cisco Routing DHCP DNS

Interconexión de las tres sedes de la empresa TecnoPyme S.L. mediante enlaces punto a punto y rutas estáticas en Cisco Packet Tracer. La sede central ofrece DHCP y DNS, combinando direccionamiento estático y dinámico.

Estructura de la red

La sede central (R1) es el núcleo y da servicios a su LAN. Las dos sucursales (R2 y R3) se conectan en serie mediante enlaces punto a punto y actúan como routers intermedios.

Objetivos

  • Configurar ruteo estático en los tres routers para conectar todas las LAN.
  • Ofrecer DHCP desde la sede central a su red local.
  • Implementar un servicio DNS bajo el dominio www.tecnopyme.com.
  • Combinar IP estática (primer equipo de cada sede) y dinámica (resto).

Ruteo estático

Cada router necesita conocer las redes que no tiene conectadas directamente. Desde R1, las redes de las sucursales se alcanzan a través de R2:

R1 · rutas estáticas
! R1 conoce las redes remotas a través de R2 (192.168.12.2)
R1(config)# ip route 192.168.20.0 255.255.255.0 192.168.12.2
R1(config)# ip route 192.168.30.0 255.255.255.0 192.168.12.2

! R3 conoce las demás redes a través de R2 (192.168.23.1)
R3(config)# ip route 192.168.10.0 255.255.255.0 192.168.23.1
R3(config)# ip route 192.168.20.0 255.255.255.0 192.168.23.1

DHCP centralizado

R1 hospeda los pools DHCP de las tres LAN. Se excluyen las primeras 20 direcciones de cada red (reservadas para equipos con IP estática). R2 y R3 reenvían las peticiones de sus LAN a R1 con ip helper-address.

R1 · DHCP
R1(config)# ip dhcp excluded-address 192.168.10.1 192.168.10.20
R1(config)# ip dhcp pool LAN_R1
R1(dhcp-config)# network 192.168.10.0 255.255.255.0
R1(dhcp-config)# default-router 192.168.10.1
R1(dhcp-config)# dns-server 192.168.40.1

! En las sucursales: reenviar el DHCP a R1
R2(config-if)# ip helper-address 192.168.10.1

Servicio DNS

El servidor de la sede central resuelve nombres internos bajo el dominio www.tecnopyme.com, y se verifica la resolución desde las tres sedes mediante pruebas de ping y consulta por nombre.

Conclusión

Un escenario clásico de WAN entre sedes que afianza el ruteo estático, el reenvío de DHCP entre redes y la resolución DNS. Comprender bien el enrutamiento manual es la mejor base antes de pasar a protocolos dinámicos y a redes en la nube.
